End-to-end DG Monitoring & Data Sharing Architecture — from physical sensors to cloud dashboard.

🔌 Sensors & Control Panel
CT clamps and analog sensors measure voltage, current, frequency, fuel level, oil pressure, and engine temperature directly from each DG set's control panel (DSE 7320).
📡 IIoT Gateway (Teltonika)
A Teltonika industrial gateway collects RS-485 / Modbus RTU signals from all sensors and forwards pre-processed data to the Raspberry Pi 5 edge node over Ethernet.
🍓 Raspberry Pi 5 Edge Node
Acts as the developer edge node — runs Python/C++ scripts, hosts a local MQTT broker, filters noise, and handles local data storage before pushing to the cloud via 4G LTE.
☁️ Cloud Platform
Receives telemetry data over 4G LTE cellular. Stores time-series records in a database, runs analytics pipelines, and fires alerts (email / SMS) for critical thresholds.
🖥️ Local PLC (Siemens S7-1200)
Optional on-site PLC for relay control, ATS logic, and SCADA integration. Communicates via Ethernet/LAN to the edge node and local workstation.
📱 Mobile / Web Dashboard
This web dashboard (and the companion mobile app) display live parameters, daily analytics, fuel consumption, and cost estimates — accessible from any device on any network.
🔄 Data Flow — How It Works
1
Generators produce data
4 DG sets run and generate electrical & mechanical data continuously.
2
Sensors capture signals
CT clamps and analog sensors read voltage, current, frequency, fuel, temp, and oil pressure.
3
Gateway converts protocols
Teltonika IIoT gateway converts Modbus RTU signals to IP packets and forwards to the edge node.
4
Edge node processes data
Raspberry Pi 5 filters noise, stores locally via MQTT broker, and sends clean data to cloud over 4G.
5
Cloud stores & analyzes
Database records every reading. Analytics engine calculates fuel consumption, cost, and efficiency.
6
Dashboard displays live data
This web dashboard refreshes every second — showing parameters, daily stats, and fuel levels.
7
Alerts fire on thresholds
Low fuel (<50 L), overload (>90%), or sensor faults trigger instant banner alerts.
8
Team takes action
Maintenance team refuels, inspects, or resets the DG based on the alert and logged data.
